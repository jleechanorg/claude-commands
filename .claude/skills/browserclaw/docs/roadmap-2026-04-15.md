### WorldAI partial capture (2026-04-15)

**4/7 endpoints captured** via superpower chrome + `fetchApi()` (HAR missed JS-triggered authenticated calls):

| Method | Endpoint | Status |
|--------|----------|--------|
| GET | `/api/campaigns` | ✅ captured via fetchApi |
| GET | `/api/campaigns/{cid}/story` | ✅ captured via fetchApi (dynamic ID) |
| GET | `/api/settings` | ✅ captured via fetchApi |
| GET | `/api/time` | ✅ captured via fetchApi |
| POST | `/api/campaigns/{id}/interaction` | ❌ blocked — OpenClaw 405 |
| GET | `/api/campaigns/{id}/export` | ❌ rate limited — 429 2/min |
| GET | `/api/campaigns/{id}` (generic) | ❌ not captured — only dynamic `{cid}/story` captured |

**HAR limitation**: HAR only captured `/api/time`. All authenticated calls made via `fetchApi()` (WorldAI's JS wrapper) were invisible to Playwright HAR recorder.

**Solution**: Superpower chrome CDP connection + `page.evaluate("window.fetchApi()")` gives full auth.

### Auth technique: Superpower Chrome CDP

```python
browser = await p.chromium.connect_over_cdp("http://localhost:9222")
context = browser.contexts[0]  # NOT browser.contexts()
page = context.pages[-1]

# Use the page's own auth-attached fetch wrapper
result = await page.evaluate("""async () => {
    return (await window.fetchApi('/api/endpoint')).data;
}""")
```

Key: The page's `fetchApi()` already has correct Firebase JWT attached — no token extraction needed.
