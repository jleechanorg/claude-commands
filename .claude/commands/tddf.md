# /tddf - Four-Layer TDD Testing Protocol (Alias)

## Purpose
Alias for `/4layer` - Implements comprehensive Test-Driven Development across 4 testing layers.

**NOTE**: This is an alias for `/4layer`. See `/4layer` for complete documentation.

## Usage
```
/tddf [feature_name]
```

## Quick Reference
- **Layer 1**: Unit Tests (isolated modules)
- **Layer 2**: Integration Tests (Python backend flow)  
- **Layer 3**: Browser Tests (mocked APIs)
- **Layer 4**: Browser Tests (real APIs)

## Execution
When `/tddf` is invoked, it executes the exact same protocol as `/4layer`:

1. **Planning Phase** (`/think`): Identify components and test scenarios
2. **Execution Phase** (`/e`): Progressive implementation through all 4 layers
3. **TDD Compliance**: Red-Green-Refactor at each layer
4. **Screenshot Documentation**: Capture UI behavior in Layers 3-4

## Why /tddf?
- **TDD**: Test-Driven Development
- **F**: Four-layer approach
- Shorter alias for frequent use
- Emphasizes TDD methodology
