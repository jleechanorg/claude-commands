# Screenshot Organization - V1 vs V2 Campaign Creation Flows

## Directory Structure

### V1 Campaign Creation Flow
- `v1_baseline/` - Original V1 implementation screenshots
- `v1_flask/` - V1 Flask backend integration evidence
- `v1_full_flow/` - Complete V1 end-to-end documentation

### V2 Campaign Creation Flow
- `v2_complete/` - Comprehensive V2 implementation evidence
  - `case1_dragon_knight/` - Dragon Knight campaign creation
  - `case2_custom_random/` - Custom campaign with random elements
  - `case3_custom_full/` - Full custom campaign creation
- `v2_react/` - React V2 frontend implementation
- `v2_current_state/` - Current implementation state
- `v2_analysis/` - Technical analysis and comparisons

### Integration & Testing
- `milestone2_integration/` - Milestone 2 integration testing screenshots
- `misc_testing/` - General UI/UX testing screenshots
- `comparison/` - Direct V1 vs V2 comparisons

## Key Flows Documented

1. **V1 Flow**: Homepage → Campaign Basics → AI Style → Launch → Character Creation (in-game)
2. **V2 Flow**: Campaign List → Multi-step Creation Wizard → Direct Game View

## Evidence Standards

All screenshots follow naming convention:
- `[version]-[case/step]-[description].png`
- Includes step numbers for sequential flows
- Documents both success and error states

## Usage

Use this organized structure to:
- Compare V1 vs V2 implementation differences
- Validate campaign creation flow completeness
- Document user experience improvements
- Support technical decision making
